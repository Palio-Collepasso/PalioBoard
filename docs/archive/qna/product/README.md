# Product Q&A

This directory contains product and domain clarifications for scope, competition behavior, lifecycle rules, scoring, visibility, and future considerations.

Use these files as follows:

- [`operators and scope.md`](operators%20and%20scope.md): supported operators, responsibilities, and product scope boundaries.
- [`competition model.md`](competition%20model.md): competition structure and how the main competitions are modeled.
- [`season setup.md`](season%20setup.md): how a season is configured, including setup expectations and constraints.
- [`game lifecycle.md`](game%20lifecycle.md): lifecycle states, transitions, and semantics after completion.
- [`scoring, jolly, and ties.md`](scoring,%20jolly,%20and%20ties.md): points, Jolly behavior, tie handling, and ranking-related rules.
- [`prepalio and giocasport.md`](prepalio%20and%20giocasport.md): specific rules and behavior for Prepalio and Giocasport.
- [`public visibility and maxi-screen.md`](public%20visibility%20and%20maxi-screen.md): what is public, what appears on the maxi-screen, and how visibility behaves.
- [`future considerations.md`](future%20considerations.md): explicitly deferred ideas and likely future extensions.
- [`open questions.md`](open%20questions.md): unresolved product questions that still need a decision.
