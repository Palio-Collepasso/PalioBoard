# Documentation Examples

Use the files in this directory as examples of how each maintained documentation type should be structured and used.

Rules for this directory:
- examples show the intended shape and level of detail for each doc type
- examples are not the canonical source of current product or operational truth
- when updating a maintained doc, compare it to the matching example before introducing new structure

Current examples:
- `business-rules.md`
- `critical-e2e-flows.md`
- `deploy.md`
- `error-contract.md`
- `fixtures.md`
- `local-dev.md`
- `runbook.md`
- `test-strategy.md`
